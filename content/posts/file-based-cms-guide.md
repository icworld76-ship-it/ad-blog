## 파일 기반 CMS란

파일 기반 CMS는 데이터베이스 대신 마크다운(.md)이나 MDX 파일을 콘텐츠 저장소로 사용하는 접근 방식입니다. 각 포스트는 하나의 파일이며, 파일 상단의 프론트매터(front matter)에 제목, 날짜, 요약 같은 메타데이터를 기록합니다.

```markdown
---
title: "포스트 제목"
date: "2026-07-01"
summary: "한 줄 요약"
---

본문 내용...
```

## 장점

- **버전 관리 용이**: Git으로 콘텐츠 변경 이력을 그대로 추적할 수 있습니다.
- **배포 간단**: 별도의 백엔드나 데이터베이스 없이 정적 사이트로 빌드 및 배포가 가능합니다.
- **이식성**: 마크다운 파일은 어떤 플랫폼으로도 쉽게 옮길 수 있습니다.

## 이 블로그의 구조

이 프로젝트는 `content/posts` 디렉터리에 있는 `.md` 파일을 `gray-matter`로 파싱하여 프론트매터와 본문을 분리하고, `remark`로 마크다운 본문을 HTML로 변환합니다.

새 글을 추가하려면 해당 디렉터리에 새 마크다운 파일을 만들고 프론트매터를 채운 뒤 본문을 작성하면 끝입니다. 별도의 코드 수정이나 데이터베이스 마이그레이션이 필요 없습니다.

## 마무리

작은 개인 블로그나 문서 사이트에는 파일 기반 CMS가 오히려 더 간단하고 빠르며 유지보수하기 쉬운 선택이 될 수 있습니다.
